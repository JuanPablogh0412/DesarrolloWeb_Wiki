No Funcionales

Num Categoria Requerimiento Descripción
1 Seguridad Autenticación segura El sistema debe implementar autenticación segura utilizando OAuth 2.0, JWT o similar para proteger las cuentas de usuario.
2 Seguridad Encriptación de datos La información sensible (contraseñas, datos de pago) debe almacenarse de forma encriptada utilizando estándares como AES o bcrypt.
3 Rendimiento Tiempo de respuesta Las solicitudes clave (autenticación, carga de propiedades) deben responder en menos de 2 segundos en condiciones normales de carga.
4 Escalabilidad Soporte para múltiples usuarios concurrentes La plataforma debe permitir la conexión simultánea de al menos 1000 usuarios sin degradación significativa del rendimiento.
5 Disponibilidad Uptime del 99.9% El sistema debe estar disponible el 99.9% del tiempo.
6 Usabilidad Interfaz intuitiva La UI debe ser accesible y fácil de usar para usuarios sin conocimientos técnicos.
7 Mantenibilidad Buenas practicas de programación El código debe seguir buenas prácticas de desarrollo y estar documentado para facilitar futuras mejoras.
8 Compatibilidad Navegadores soportados La plataforma debe ser compatible con las versiones más recientes de Chrome, Firefox, Edge y Safari.
